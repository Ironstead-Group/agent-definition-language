# IETF RFC Track

## Overview

- **Organization:** Internet Engineering Task Force (IETF)
- **Output:** Internet-Drafts (I-Ds), RFCs (Informational or Standards Track).
- **Relevance to ADL:** Formal, stable specification for the agent definition format and semantics; good for normative references in protocols and implementations.

## Links

- [IETF Datatracker](https://datatracker.ietf.org/)

- [Guidelines for Authors](https://www.rfc-editor.org/rfc-editor-guide/)

## Submission Requirements (Summary)

- **Format:** Internet-Draft in XML (xml2rfc) or Markdown (kramdown-rfc).
- **Copyright:** IETF Trust; contributors grant license (see Trust Legal Provisions).
- **Process:** Submit I-D; find appropriate WG (or Individual Submission); address feedback; IESG review; RFC Editor publication.
- **IPR:** Declare compliance with BCP 78/79; optional patent disclosure.

## Notes

- Decide whether ADL is Informational (description of format) or Standards Track (normative standard). Informational is often easier for a first submission.
- Consider a dedicated WG (e.g., “agent” or “ai”) or adoption by an existing WG (e.g., dispatch, or a new area).
- Reuse of existing formats (YAML/JSON, naming) should be cited (e.g., OpenAPI, JSON Schema).
